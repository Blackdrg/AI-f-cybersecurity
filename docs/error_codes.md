# Error Code Catalog

This document lists all error codes used by the AI-f API, their meanings, and typical causes.

## HTTP Status Codes

The API uses standard HTTP status codes to indicate the outcome of requests.

| Code | Name | Description |
|------|------|-------------|
| 200 | OK | The request succeeded. |
| 201 | Created | The request succeeded and a new resource was created. |
| 204 | No Content | The request succeeded but there is no content to return. |
| 400 | Bad Request | The request was malformed or missing required parameters. |
| 401 | Unauthorized | Authentication is required or has failed. |
| 403 | Forbidden | The authenticated user does not have permission to perform the requested action. |
| 404 | Not Found | The requested resource does not exist. |
| 409 | Conflict | The request could not be completed due to a conflict with the current state of the resource. |
| 422 | Unprocessable Entity | The request was well-formed but contains semantic errors (validation errors). |
| 429 | Too Many Requests | The user has sent too many requests in a given amount of time (rate limiting). |
| 500 | Internal Server Error | An unexpected condition was encountered on the server. |
| 502 | Bad Gateway | The server, while acting as a gateway or proxy, received an invalid response from the upstream server. |
| 503 | Service Unavailable | The server is currently unable to handle the request due to temporary overload or maintenance. |
| 504 | Gateway Timeout | The server, while acting as a gateway or proxy, did not receive a timely response from the upstream server. |

## Application-Specific Error Responses

When an error occurs, the API returns a JSON response with the following structure:

```json
{
  "detail": [
    {
      "loc": ["string"],
      "msg": "string",
      "type": "string"
    }
  ]
}
```

This is the standard format for HTTP 422 (Unprocessable Entity) errors generated by request validation (via Pydantic/FastAPI). Other error codes may return a simpler JSON object or an empty body.

### Validation Error Details (HTTP 422)

Each validation error object in the `detail` array contains:

- `loc`: A list indicating the location of the error (e.g., `["body", "field_name"]` for a body field, or `["query", "param_name"]` for a query parameter).
- `msg`: A human-readable error message.
- `type`: The type of error (e.g., `value_error`, `type_error.integer`, `missing`).

### Common Validation Errors

| Error Type | Message Example | Location Example | Meaning |
|------------|-----------------|------------------|---------|
| `missing` | Field required | `["body", "email"]` | A required field is missing. |
| `type_error.integer` | value is not a valid integer | `["query", "top_k"]` | The value provided is not an integer. |
| `value_error` | Invalid image format | `["body", "image"]` | The uploaded file is not a valid image. |
| `value_error.any_str.min_length` | ensure this value has at least 1 characters | `["body", "name"]` | The string is too short. |
| `value_error.any_str.max_length` | ensure this value has at most 255 characters | `["body", "name"]` | The string is too long. |
| `type_error.bool` | Input should be a valid boolean | `["body", "consent"]` | The value is not a valid boolean. |
| `value_error.image` | Invalid image dimensions | `["body", "image"]` | The image dimensions are not supported. |
| `value_error.file.ext` | Unsupported file extension | `["body", "voice_file"]` | The file extension is not allowed. |

### Authentication and Authorization Errors

| Code | Message | Meaning |
|------|---------|---------|
| 401 | Could not validate credentials | The JWT token is missing, invalid, or expired. |
| 401 | Invalid authentication token | Same as above. |
| 403 | Insufficient permissions | The user's role does not allow the requested action. |
| 403 | Organization access denied | The user does not belong to the specified organization. |

### Rate Limiting Errors

| Code | Headers | Meaning |
|------|---------|---------|
| 429 | `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` | Too many requests. See headers for limits and reset time. |

### Resource Conflict Errors

| Code | Message | Meaning |
|------|---------|---------|
| 409 | Person with this email already exists | Attempting to create a user with an existing email. |
| 409 | Organization name already taken | Attempting to create an organization with a duplicate name. |
| 409 | Camera with this RTSP URL already exists in organization | Duplicate camera registration. |

### Server Errors

| Code | Message | Meaning |
|------|---------|---------|
| 500 | Internal server error | Unexpected error on the server. Check logs for details. |
| 502 | Bad gateway | Error communicating with an upstream service (e.g., model service, database). |
| 503 | Service unavailable | The server is temporarily unable to handle the request (e.g., database connection pool exhausted, model not loaded). |
| 504 | Gateway timeout | The server did not receive a timely response from an upstream service. |

## Error Handling Guidelines

### For Clients

1. **Check HTTP status code first** to determine the general outcome.
2. **For 422 errors**, parse the `detail` array to understand validation errors and display them to the user.
3. **For 401 errors**, redirect to login or refresh token if possible.
4. **For 403 errors**, inform the user they lack permission.
5. **For 429 errors**, respect the `Retry-After` header if present, or use the `X-RateLimit-Reset` header to determine when to retry.
6. **For 5xx errors**, consider retrying with exponential backoff, but avoid aggressive retries.

### For Developers

- Use FastAPI's HTTPException to return appropriate status codes.
- For validation errors, let FastAPI handle them automatically (they return 422).
- Avoid exposing internal error details in 500 responses in production; log them internally.
- Define custom exception handlers for application-specific errors if needed.

### Validation Error (422)

```json
{
  "detail": [
    {
      "loc": [
        "body",
        "email"
      ],
      "msg": "value is not a valid email address",
      "type": "value_error.email"
    },
    {
      "loc": [
        "body",
        "password"
      ],
      "msg": "ensure this value has at least 8 characters",
      "type": "value_error.any_str.min_length"
    }
  ]
}
```

### Unauthorized (401)

```json
{
  "detail": "Not authenticated"
}
```

### Forbidden (403)

```json
{
  "detail": "Insufficient permissions"
}
```

```json
{
  "detail": "Rate limit exceeded"
}
```
